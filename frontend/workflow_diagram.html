Module 2: Timbre Watermarking (Spectral Embedding)

🎼 What is Timbre?

Timbre is the quality that distinguishes sounds of the same pitch and volume - it's what makes a guitar sound different from a piano playing the same note. Timbre is determined by the audio's spectral (frequency) characteristics.

Our watermarking technique embeds data by subtly modifying the timbre/spectral profile of the audio signal using FFT-based frequency domain manipulation. This approach is imperceptible to human ears but machine-detectable.

Why Timbre-Based Watermarking?

  • Imperceptibility: Small changes in spectral energy are inaudible to humans
  • Robustness: Frequency domain modifications survive audio compression
  • Capacity: Multiple frequency bins allow sufficient data embedding

Timbre Encoding: Push-Pull Strategy

The watermark is encoded by adjusting the relative energy between two frequency bin groups, effectively modifying the timbre profile:

TARGET BINS [200-206] ≈ 1.5 kHz (Low-mid frequencies) NEIGHBOR BINS [220-226] ≈ 3 kHz (Mid frequencies) Bit = 1: Target = HIGH energy, Neighbor = LOW energy → Boost Target Timbre Bit = 0: Target = LOW energy, Neighbor = HIGH energy → Suppress Target Timbre

This push-pull approach modifies the spectral signature (timbre) of each audio chunk, encoding binary data while maintaining perceived audio quality.

Module 3: Timbre Watermark Extraction

Extraction reverses the embedding process by analyzing the timbre profile (frequency spectrum) of each chunk and using majority voting across redundant copies for reliability.

Key Innovations

  • Dual Protection System: Combines AI-based detection with Timbre watermarking
  • Timbre-Based Spectral Embedding: Hides data in frequency bins (timbre) for imperceptibility
  • Reed-Solomon Error Correction: Tolerates up to 2-byte errors per codeword
  • 5x Redundancy with Majority Voting: Ensures reliable extraction
  • Real-time Processing: Flask API enables instant analysis

Algorithm Summary

MFCC Feature Extraction

Audio → 22050 Hz Resampling → MFCC (20 coefficients) → Mean → 20-dim Vector

Timbre Watermark Encoding

Text → UTF-8 → Pad 11 bytes → RS Encode → 15 bytes → 120 bits → 5x = 600 bits

Timbre Embedding (FFT)

Audio[1024 samples] → FFT → Modify Bins[200-206, 220-226] → IFFT → Timbre-Modified Audio
